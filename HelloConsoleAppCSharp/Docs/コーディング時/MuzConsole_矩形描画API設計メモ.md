# MuzConsole 矩形描画 API 設計メモ

`MuzConsole` の矩形描画まわりの仕様を整理するためのメモです。  
このプロジェクトは、人間が学習しやすいことを優先するため、API の挙動をできるだけ単純にします。  


## 対象 API

- `FillRectAsync(...)`
- `DrawDoubleBorderRectAsync(...)`


## 設計方針

- 引数の意味を単純にする
- 負の値で逆方向へ描画する仕様は持ち込まない
- 潰れてしまうサイズは無理にサポートしない
- 学習用として、説明しやすい挙動を優先する


## FillRectAsync の仕様

### 役割

指定した左上座標から、矩形領域を塗りつぶします。  

### 引数の意味

- `left`: 左上の X 座標
- `top`: 左上の Y 座標
- `width`: 横幅
- `height`: 高さ

### 仕様

- `width <= 0` のときは何も描画しません
- `height <= 0` のときも何も描画しません
- 負の幅や高さで、反対方向へ塗りつぶす仕様はサポートしません

### 意図

`FillRectAsync(left, top, width, height)` は、
**左上から右下へ広がる矩形** として扱います。  

そのため、
- 左へ広がる
- 上へ広がる
といった挙動は考えません。  


## DrawDoubleBorderRectAsync の仕様

### 役割

指定した左上座標から、二重線の枠付き矩形を描画します。  

### 引数の意味

- `left`: 左上の X 座標
- `top`: 左上の Y 座標
- `width`: 横幅
- `height`: 高さ

### 仕様

- `width < 2` のときは何も描画しません
- `height < 2` のときも何も描画しません
- 二重線の矩形が潰れてしまうサイズはサポートしません

### 意図

二重線の枠付き矩形は、最低でも
- 左上
- 右上
- 左下
- 右下
の 4 つの角と、辺を表現できるサイズが必要です。  

サイズが足りない場合に無理に描画すると、
API の挙動が分かりにくくなります。  
そのため、このプロジェクトでは **何も描画しない** ことにします。  


## この方針のメリット

- API の説明が簡単
- 呼び出し側が挙動を予想しやすい
- 逆方向描画の分岐を実装しなくてよい
- 教材として読みやすい


## 今後の検討項目

- 画面外にはみ出したときの扱い
- クリッピングするかどうか
- 単線の矩形 `DrawSingleBorderRectAsync(...)` を追加するか
- `ClearRectAsync(...)` を追加するか


## ひとまずの結論

このプロジェクトの矩形描画 API は、次の単純な仕様で進めます。  

- `FillRectAsync(...)` は `width <= 0` または `height <= 0` のとき何もしない
- `DrawDoubleBorderRectAsync(...)` は `width < 2` または `height < 2` のとき何もしない
- 逆方向描画や、潰れるサイズの特殊対応はしない
