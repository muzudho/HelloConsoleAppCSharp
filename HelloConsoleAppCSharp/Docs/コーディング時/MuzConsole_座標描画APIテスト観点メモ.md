# MuzConsole 座標描画 API テスト観点メモ

`MuzConsole` の座標描画まわりの API について、今の仕様を崩さないためのテスト観点を整理するメモです。  
主に次の API を対象にします。  

- `WriteAtAsync(...)`
- `WriteLineAtAsync(...)`
- `FillRectAsync(...)`
- `DrawDoubleBorderRectAsync(...)`


## テスト方針

このプロジェクトでは、コンソール描画を人間向けに学習しやすい形でまとめています。  
そのため、厳密な内部実装の差ではなく、**利用者から見た挙動** を確認する観点を優先します。  

確認したい主な軸は次のとおりです。  

- 例外にならないか
- no-op 条件が守られているか
- 必要ならバッファーが自動拡張されるか
- カーソル位置が元に戻るか
- 改行あり／なしの違いが保たれているか


## WriteAtAsync のテスト観点

### 正常系

- バッファー内の座標に 1 行の文字列を書ける
- 前景色、背景色を指定して書ける
- 空文字列でも例外にならない
- メッセージ内に改行があっても例外にならない

### 仕様確認

- **改行しない** こと
- 描画後にカーソル位置が元に戻ること
- 出力がリダイレクトされていない場合、必要に応じてバッファーが広がること

### 異常系・境界値

- `left < 0` のときの挙動
- `top < 0` のときの挙動
- 非常に長い 1 行文字列を渡したときの挙動
- 複数行文字列を渡したときの必要高さ計算


## WriteLineAtAsync のテスト観点

### 正常系

- バッファー内の座標に 1 行の文字列を書ける
- 前景色、背景色を指定して書ける
- 空文字列でも例外にならない
- メッセージ内改行を含む文字列でも例外にならない

### 仕様確認

- **表示後に改行する** こと
- `WriteAtAsync(...)` と違って改行ありであること
- 描画後にカーソル位置が元に戻ること
- 出力がリダイレクトされていない場合、必要に応じてバッファーが広がること

### 異常系・境界値

- `left < 0` のときの挙動
- `top < 0` のときの挙動
- メッセージ末尾改行ぶんの高さ確保が足りるか
- 複数行文字列で高さ計算が破綻しないか


## FillRectAsync のテスト観点

### 正常系

- 指定した座標・幅・高さで矩形領域を塗りつぶせる
- 色付き版でも描画できる
- 描画後にカーソル位置が元に戻る
- 出力がリダイレクトされていない場合、必要に応じてバッファーが広がる

### no-op 条件

- `width == 0` のとき何もしない
- `width < 0` のとき何もしない
- `height == 0` のとき何もしない
- `height < 0` のとき何もしない

### 異常系・境界値

- `left < 0` のときの挙動
- `top < 0` のときの挙動
- `width == 1`、`height == 1` の最小有効サイズ
- バッファーぎりぎりの座標での描画


## DrawDoubleBorderRectAsync のテスト観点

### 正常系

- 指定した座標に二重線の矩形を描ける
- 描画後にカーソル位置が元に戻る
- 出力がリダイレクトされていない場合、必要に応じてバッファーが広がる

### no-op 条件

- `width == 1` のとき何もしない
- `width < 1` のとき何もしない
- `height == 1` のとき何もしない
- `height < 1` のとき何もしない

### 境界値

- `width == 2`、`height == 2` の最小有効サイズ
- `width == 2`、`height > 2` の縦長矩形
- `width > 2`、`height == 2` の横長矩形

### 異常系

- `left < 0` のときの挙動
- `top < 0` のときの挙動


## 自動バッファ拡張のテスト観点

### 文字列描画

- `WriteAtAsync(...)` で右端に近い位置へ長い文字列を書いても落ちない
- `WriteLineAtAsync(...)` で下端に近い位置へ複数行文字列を書いても落ちない
- 必要最小限以上にバッファーが縮まないこと

### 矩形描画

- `FillRectAsync(...)` で右下方向にバッファー外まではみ出す矩形でも落ちない
- `DrawDoubleBorderRectAsync(...)` で右下方向にバッファー外まではみ出す矩形でも落ちない

### リダイレクト時

- `Console.IsOutputRedirected == true` 相当の状況では、無理にバッファー操作しないこと


## カーソル退避のテスト観点

対象:

- `WriteAtAsync(...)`
- `WriteLineAtAsync(...)`
- `FillRectAsync(...)`
- `DrawDoubleBorderRectAsync(...)`

確認事項:

- 描画前の `Console.CursorLeft`
- 描画前の `Console.CursorTop`
- 描画後にその位置へ戻ること


## 手動確認でもよい項目

コンソール API は環境差が出やすいため、次の項目は手動確認でも価値があります。  

- バッファー拡張後の見た目
- 長文の折り返し方
- 日本語や全角文字を含むときの見え方
- 二重線文字の見え方


## ひとまず優先するテスト

最初は次の観点だけでも押さえると、かなり安心です。  

1. `WriteAtAsync(...)` は改行しない
2. `WriteLineAtAsync(...)` は改行する
3. `FillRectAsync(...)` は `width <= 0 || height <= 0` で no-op
4. `DrawDoubleBorderRectAsync(...)` は `width < 2 || height < 2` で no-op
5. 座標が右下にはみ出すケースで、バッファー自動拡張により落ちない
6. 描画後にカーソル位置が戻る


## 補足

コンソール描画は、完全に自動テストしにくい部分もあります。  
そのため、このプロジェクトでは

- 仕様メモ
- 手動確認用コマンド
- 必要最低限の自動テスト

を組み合わせる考え方が向いています。
