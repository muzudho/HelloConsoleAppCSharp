# アプリケーション設定ファイルの準備

Microsoft 公式の JSON 形式の設定ファイルを使用できるよう準備します。  
（例えば 2026年3月現在、TOML 形式の設定ファイルはサポートされていません）  


## ［アプリケーション設定ファイル］作成

ファイル名は `appsettings.json` とし、プロジェクトルートに置いてください。  

📄 `appsettings.json`:  

```json
{
    "AppName": "HelloConsoleAppCSharp"
}
```

JSONファイルの内容の構成は、`Logging` や `ConnectionStrings` セクションなどいくつかについては、Microsoft 公式のドキュメントにある構成を参考にしてください。  
それ以外は自由です。  


## ［アプリケーション設定ファイル］をプロジェクトへ追加

Visual Studio 2026 の ソリューションエクスプローラーから `appsettings.json` 項目を右クリック。  
`プロパティ` を選択。  
`ビルドアクション` を `コンテンツ` に変更。  
`出力ディレクトリにコピー` を `常にコピーする` に変更。  

※ 以上により、ビルド後の出力ディレクトリ（`bin/Debug/net8.0/` など）に `appsettings.json` がコピーされるようになります。  


## パッケージのインストール

NuGet を使って以下のパッケージをインストールしてください。  

* `Microsoft.Extensions.Configuration` - アプリケーションの設定を管理するためのパッケージです。
* `Microsoft.Extensions.Configuration.Json` - JSON 形式の設定ファイルを読み込むためのパッケージです。


## ファイル作成

📄 `MuzAppSettings.cs`:  

```csharp
namespace HelloConsoleAppCSharp.Infrastructure.Configuration;

/// <summary>
///     <pre>
/// appsettings.json ファイルのオブジェクト。
///     </pre>
/// </summary>
internal class MuzAppSettings
{
    /// <summary>
    /// アプリケーション名。
    /// </summary>
    public string AppName { get; set; } = string.Empty;
}
```

👆 ファイルの内容の例（＾～＾）  


📄 `MuzAppSettingsHelper.cs`:  

```csharp
namespace HelloConsoleAppCSharp.Infrastructure.Configuration;

using Microsoft.Extensions.Configuration;   // `.AddJsonFile()` とか `IConfiguration` とかのために必要（＾～＾）
using Microsoft.Extensions.DependencyInjection; // `builder.Services.Configure<T>()` のために必要（＾～＾）
using Microsoft.Extensions.Hosting; // `HostApplicationBuilder` のために必要（＾～＾）

/// <summary>
/// ［アプリケーション設定ファイル］そのもののコードではないが、それを使う準備をするためのコードをまとめたクラスだぜ（＾～＾）！
/// </summary>
internal static class MuzAppSettingsHelper
{
    /// <summary>
    /// ［ホストビルド］の前に、諸々の設定をするぜ（＾～＾）
    /// </summary>
    public static void SetupBeforeHostBuild(IHostApplicationBuilder builder)
    {
        // ［設定ファイル］の設定（＾～＾）
        builder.Configuration
            .AddJsonFile("appsettings.json", optional: false, reloadOnChange: true)  // 必須ファイル。ファイル編集後にはリロードするぜ（＾～＾）！
            .AddJsonFile($"appsettings.{builder.Environment.EnvironmentName}.json", optional: true, reloadOnChange: true)   // 環境ごとのファイル。ファイル編集後にはリロードするぜ（＾～＾）！　必須じゃないぜ（＾～＾）！
            .AddEnvironmentVariables();  // 環境変数で［環境の名前］を使って、設定のカスケード（上書き）を可能にするぜ（＾～＾）

        // ここで DI コンテナにサービスを登録（＾～＾）
        builder.Services.Configure<MuzAppSettings>(builder.Configuration);  // さっきの［設定ファイルの設定］を MuzAppSettings クラスにバインド（＾～＾）
    }
}
```

👆 ファイルの内容の例（＾～＾）  


## 組み立て

📄 `MuzInfrastructureService.cs`:  

```csharp
namespace HelloConsoleAppCSharp;

using HelloConsoleAppCSharp.Infrastructure.Configuration;

internal class MuzInfrastructureHelper
{
    /// <summary>
    /// ホストビルドする前にやることがあればここでやるぜ（＾～＾）！例えば、［サービス］を追加したりとか、そういうのだぜ（＾～＾）！
    /// </summary>
    /// <param name="builder"></param>
    /// <returns></returns>
    private static async Task SetupBeforeBuildAsync(IHostApplicationBuilder builder)
    {
        // お前のアプリケーションに合わせて、［サービス］を追加していってくれだぜ（＾～＾）！
        Console.WriteLine("ホストビルドする前にやることがあればここでやるぜ（＾～＾）！例えば、［サービス］を追加したりとか、そういうのだぜ（＾～＾）！");

        //
        // ［アプリケーション設定ファイル］サービスの登録
        //
        MuzAppSettingsHelper.SetupBeforeHostBuild(builder);
    }
}
```

👆 `HelloConsoleAppCSharp/MuzInfrastructureService.cs` ファイルの `MuzInfrastructureService.SetupBeforeBuildAsync()` メソッドの中で、`MuzAppSettingsHelper.SetupBeforeHostBuild(builder)` を呼び出してください。


## 動作確認

📄 `HelloConsoleAppCSharp/Program.cs`:  

```csharp
using HelloConsoleAppCSharp.Infrastructure.Configuration;

// 略

    await MuzInfrastructureHelper.BuildHostAsync(
        commandLineArgs: args,
        onHostEnabled: async (services) =>
        {
            // ここから［サービス・プロバイダー］（services）が使えるぜ（＾▽＾）！
            Console.WriteLine("ここから［サービス・プロバイダー］（services）が使えるぜ（＾▽＾）！");

            // ［設定ファイル］の動作確認してみようぜ（＾～＾）
            var appSettings = services.GetRequiredService<IOptions<MuzAppSettings>>().Value;
            Console.WriteLine($"AppName: {appSettings.AppName}");
        });
```

実行して、確認してください。  
以上で、［アプリケーション設定ファイルの準備］は完了です。
