# ロギングの準備＜その２＞（複数のロガーの使い分け）


## パッケージのインストール

NuGet を使って以下のパッケージをインストールしてください。  

* `Serilog.Extensions` - （使うにはライセンス条項の同意が必要）ロガーを複数使い分けるときに `SourceContext = 'MuzOthersLogger'` のような式を有効にするために必要。

```json
{

                                    }
                                }
                            ],
                            "WriteTo": [
                                {
                                    "Name": "File",
                                    "Args": {

                                    }
                                }
                            ]

            }
        }

```

👆 `WriteTo` セクションが配列になっていて、`Console` と `File` という名前を持つセクションが並んでいるところに、［`Logger` という名前を持つセクション］を複数追加してください。  

［`Logger` という名前を持つセクション］は、`Args/configureLogger/Filter/Args/expression` を使って、識別するようにします。  
この部分は本来、クラスの名前空間を部分一致させるものですが、今回は単に、［カテゴリー名］での一致でロガーを選ぶようにしています。

{


    // ========================================
    // 窓口プロパティ
    // ========================================


    /// <summary>

    /// </summary>

    /// 大量のロガー。

📄 `HelloConsoleAppCSharp/Infrastructure/Logging/MuzLoggingService.cs`:  

```csharp
namespace HelloConsoleAppCSharp.Infrastructure.Logging;

using Microsoft.Extensions.Logging;

internal class MuzLoggingService : IMuzLoggingService
{


    // ========================================
    // 生成／破棄
    // ========================================


    /// <summary>
    /// 
    /// </summary>
    /// <param name="loggerFactory">DI から取る</param>
    public MuzLoggingService(ILoggerFactory loggerFactory)
    {
        // ［ロガー］を分ける動作確認してみようぜ（＾～＾） ［カテゴリー名］でロガー作成（ここがポイント！）
        this.Others = loggerFactory.CreateLogger("MuzOthersLogger");    // ［その他のログ用］（＾～＾）
        this.Verbose = loggerFactory.CreateLogger("MuzVerboseLogger");  // ［大量のログ用］（＾～＾）
    }


    // ========================================
    // 窓口プロパティ
    // ========================================


    public ILogger Others { get; init; }

    public ILogger Verbose { get; init; }
}
```
