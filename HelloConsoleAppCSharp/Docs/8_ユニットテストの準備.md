# ユニットテストの準備


## 事前確認

ユニットテストは、専用のプロジェクトを作る必要があります。  
（コードと、テストを同じプロジェクトに入れた運用は非現実的です）  

フォルダー構成は、主に以下の２つのどちらかかと思います。  

（１） Microsoft Visual Studio の従来の方法  

```plain
📁 ソリューション　（＝リポジトリー）
+-- 📁 プロジェクト
```

👆 （１）の方法では、ソリューションが１つ、プロジェクトが１～複数入っています。  


（２） Microsoft Visual Studio の新しい方法 

```plain
📁 リポジトリー
+-- 📁 ソリューション
+-- 📁 プロジェクト
```

👆 （２）の方法では、ソリューションの名前も、プロジェクトの名前も `HelloConsoleAppCSharp` で、プロジェクトは１つしかありません。  

２つとも説明するのは大変なので、（２）として説明します。  
例えば、以下（抜粋）のようになっているとします。  

```plain
📁 HelloConsoleAppCSharp
+-- 📄 HelloConsoleAppCSharp.slnx		- ソリューション・ファイル
+-- 📄 HelloConsoleAppCSharp.csproj		- プロジェクト・ファイル
+-- 📄 Program.cs						- プログラムのエントリー・ポイント
```


## 新規ソリューション、新規プロジェクトの作成

* `HelloConsoleAppCSharpTests` という名前のプロジェクト（ソリューション）を新規作成してください。  
* プロジェクトの種類は `xUnit テスト プロジェクト` を選択してください。
    * `C#`, `Windows`, `xUnit` でフィルターをかけると見つけやすいと思います。


## パッケージのアンインストール

古いパッケージがインストールされています。  
NuGet を使って、以下のパッケージをアンインストールしてください。

* `xunit` - xUnit (2.9.3)。
* `Microsoft.NET.Test.Sdk`


## パッケージのインストール


NuGet を使って `HelloConsoleAppCSharpTests` プロジェクトに、以下のパッケージをインストールしてください。  

* `xunit.v3` - xUnit.net v3。
* `xunit.v3.runner.visualstudio` - （既にインストールされているかも？）Visual Studio で xUnit.net v3 を走らせるためのランナー。


## テストのフォルダー（名前空間）の作成

とりあえず `HelloConsoleAppCSharpTests` プロジェクトに、以下のファイルを作成してください。  

📄 `HelloConsoleAppCSharpTests/ExampleUnitTests.cs` ファイル:  

```csharp
namespace HelloConsoleAppCSharpTests;

using Xunit;

/// <summary>
///     <pre>
/// 単体テストの例。
/// 
///     - Grok の書いたものを調整しました。
///     </pre>
/// </summary>
public class ExampleUnitTests
{
    /// <summary>
    /// 2 と 3 を足したら 5 になることをテストする例。
    /// </summary>
    [Fact]
    public void Add_TwoPlusThree_ReturnsFive()
    {
        // Arrange
        var calc = new Calculator();

        // Act
        int result = calc.Add(2, 3);

        // Assert
        Assert.Equal(5, result);
    }


    /// <summary>
    ///     <pre>
    /// データ駆動テスト。
    /// 
    ///     - 複数のケースを一気にやる例。
    ///     </pre>
    /// </summary>
    /// <param name="a"></param>
    /// <param name="b"></param>
    /// <param name="expected"></param>
    [Theory]
    [InlineData(2, 3, 5)]
    [InlineData(0, 0, 0)]
    [InlineData(-5, 3, -2)]
    public void Add_VariousInputs_ReturnsExpected(int a, int b, int expected)
    {
        var calc = new Calculator();
        int result = calc.Add(a, b);
        Assert.Equal(expected, result);
    }


    /// <summary>
    /// 0 で割ろうとしたら例外が投げられることをテストする例。
    /// </summary>
    [Fact]
    public void Divide_ByZero_ThrowsException()
    {
        var calc = new Calculator();
        Assert.Throws<DivideByZeroException>(() => calc.Divide(10, 0));
    }
}


/// <summary>
/// テスト対象クラス（実際はメインのプロジェクト側に置く）。
/// </summary>
public class Calculator
{
    /// <summary>
    /// x + y をします。
    /// </summary>
    /// <param name="x"></param>
    /// <param name="y"></param>
    /// <returns></returns>
    public int Add(int x, int y) => x + y;


    /// <summary>
    /// x / y をします。y が 0 のときは例外を投げます。
    /// </summary>
    /// <param name="x"></param>
    /// <param name="y"></param>
    /// <returns></returns>
    /// <exception cref="DivideByZeroException"></exception>
    public int Divide(int x, int y) => y == 0
        ? throw new DivideByZeroException()
        : x / y;
}
```


## プロジェクトの種類を exe にする

ソリューション・エクスプローラーで、`HelloConsoleAppCSharpTests` プロジェクトを右クリックして、［プロパティ］を選択してください。  
プロジェクトのプロパティが開いたら、［アプリケーション］タブを選択して、［出力の種類］を `コンソール アプリケーション` にしてください。  


## ソリューションのリビルド

`HelloConsoleAppCSharpTests` プロジェクトの Visual Studio 2026 のメインメニューから［ビルド　＞　ソリューションのリビルド］をクリックしてください。  

* もしあとで上手く動かなかったときは、Visual Studio 2026 を終了し、 `.vs`, `bin`, `obj` フォルダーを削除してから、もう一度 Visual Studio 2026 を起動して、ソリューションのリビルドをしてみてください。  


## テストの実行（動作確認）

`HelloConsoleAppCSharpTests` プロジェクトの Visual Studio 2026 のメインメニューから［テスト　＞　すべてのテストを実行］をクリックしてください。  
テスト・エクスプローラーに、自動的にテストが探し出されて、表示されるはずです。  

あとは、テスト・エクスプローラーの使い方を調べて、使ってください。頑張ってください。  


## 既存プロジェクトの追加

* `HelloConsoleAppCSharpTests` ソリューションに、既存のプロジェクト（例えば `HelloConsoleAppCSharp` プロジェクト）を追加してください。  
* `HelloConsoleAppCSharpTests` プロジェクトを右クリックして、［追加　＞　プロジェクトの参照］をクリックし、既存のプロジェクトを選択してください。

あとは、テストコードの中で、既存プロジェクトのクラスを使って、テストコードを書いてください。  

以上で、［ユニットテストの準備］は完了です。
